## Objective
建立个人写作风格库，支持：
1. 分析文本提取风格指纹
2. 保存风格模板供复用
3. 快速切换不同写作风格
4. 为 news-content-factory 提供风格配置

## Available Styles（预设模板）

### style-deep-analysis（深度分析型）
**来源**：模仿财新、第一财经、FT中文
**适用**：财经分析、政策解读、行业研究

**风格指纹**：
- **句式**：长短结合，复杂从句，平均句长35字
- **语气**：客观、冷静、数据驱动、权威感
- **人称**：第三人称为主，偶尔"我们"
- **过渡**："值得注意的是"、"从更深层次看"、"数据显示"
- **段落**：4-5行，信息密度高，逻辑严密
- **修辞**：类比、历史对比、数据论证
- **标点**：规范使用分号、冒号，少用感叹号

**示例片段**：
> 从宏观层面观察，这一现象并非孤立事件。回溯过去五年的政策脉络，我们可以发现类似的调控逻辑在2019年和2021年都曾出现，但此次的力度和广度显然超出了市场预期。数据显示，今年前两个月相关指标已较去年同期下滑23%，这一降幅在近十年中仅次于2008年金融危机时期。

---

### style-casual-interpretation（轻松解读型）
**来源**：模仿虎嗅、36氪、 latepost
**适用**：科技动态、产品分析、行业八卦

**风格指纹**：
- **句式**：短句为主，15-25字，口语化
- **语气**：轻松、幽默、网络化、年轻化
- **人称**："你"、"我"频繁使用，拉近距离
- **过渡**："说白了"、"有意思的是"、"更关键的是"、"你发现没"
- **段落**：2-3行，大量留白，适合手机阅读
- **修辞**：网络热词、表情包文字、设问句
- **标点**：爱用括号（补充）、破折号、偶尔~波浪号

**示例片段**：
> 说白了，这次更新就是冲着竞争对手去的。
> 
> 你发现没？从去年开始，整个行业都在卷这个功能。但有意思的是，大厂们似乎集体忽略了一个更本质的问题——用户真的需要吗？
> 
> 答案可能有点扎心。

---

### style-personal-essay（个人随笔型）
**来源**：模仿顶级公众号大V、知识星球KOL
**适用**：观点表达、个人成长、情感共鸣

**风格指纹**：
- **句式**：长短变化大，根据情绪调整
- **语气**：真诚、有温度、略带感性、思考型
- **人称**：第一人称"我"为主，分享个人经历
- **过渡**："那天"、"后来"、"说实话"、"我开始意识到"
- **段落**：根据叙事节奏，短则1行，长则5-6行
- **修辞**：故事开场、细节描写、金句收尾、反问引发思考
- **标点**：省略号...营造停顿，感叹号表达情绪

**示例片段**：
> 那天凌晨三点，我盯着屏幕上的数据，突然意识到一个问题。
> 
> 过去三年，我一直以为自己在做正确的事。但那一刻，我开始怀疑——如果方向错了，努力还有什么意义？
> 
> 后来我才明白，成长不是线性的。那些看似浪费的时间，其实在悄悄塑造你。

---

### style-international-perspective（国际视野型）
**来源**：模仿经济学人、华尔街日报、Foreign Affairs
**适用**：国际政治、全球经济、跨国比较

**风格指纹**：
- **句式**：长句、复合句，逻辑层次多
- **语气**：精英化、全球视角、历史纵深感
- **人称**：中性，偶尔"我们"指代西方/全球
- **过渡**："从历史长河看"、"相比之下"、"不容忽视的是"
- **段落**：5-6行，论证充分，引用丰富
- **修辞**：历史对比、跨国案例、专家引用
- **词汇**：专业术语、外来语、学术化表达

**示例片段**：
> 若将视野拉长至冷战结束后的三十年，当前的局势呈现出一种令人不安的熟悉感。与1970年代的石油危机不同，此次的挑战更多源于制度层面的竞争而非单纯的市场波动。正如某位资深外交官所言，"我们正处在一个范式转移的前夜"，而这一判断正在获得越来越多的政策制定者认同。

---

### style-sharp-commentary（犀利评论型）
**来源**：模仿自媒体意见领袖、独立评论员
**适用**：热点评论、观点交锋、争议话题

**风格指纹**：
- **句式**：短句、排比句、反问句，节奏快
- **语气**：观点鲜明、不怕得罪人、战斗感、情绪强烈
- **人称**："我"鲜明立场，直接点名批评
- **过渡**："但问题是"、"真相是"、"说白了"、"别被忽悠了"
- **段落**：1-3行，冲击力强，留白制造节奏
- **修辞**：排比、反问、对比、讽刺、金句密集
- **标点**：感叹号！反问号？破折号——频繁使用

**示例片段**：
> 别再自欺欺人了。
> 
> 真相是什么？真相就是——既得利益者不想改变，而普通人还在做梦。
> 
> 你看那数据，23%的降幅，这是"调整"吗？这是崩塌！
> 
> 更可笑的是什么？是还有人替他们洗地。

---

### style-moomoo-casual（理智随性型）

**来源**：模仿猫笔刀（moomoocat）
**适用**：理财随笔、交易心得、科技趋势、社会逻辑分析

**风格指纹**：
  - **人称与视角**：第一人称“我”为主。定位是“冷眼旁观的互联网原住民”，坦诚分享个人逻辑，不教人做事，
    只分享思路。
  - **句式与节奏**：以大白话短句为主，穿插口语化的长句。追求“说话的节奏感”，多用“说白了”、“挺有意思
    的”、“出了点小插曲”等转折词。
  - **核心语调** ：平实、理智、随性。拒绝夸张和煽情（严禁使用“残酷”、“重磅”、“惊天”等词）。有一种“收
    市后喝杯咖啡聊两句”的松弛感。
  - **论述逻辑**：
      - 由逻辑谬误切入：常用“这个逻辑最吊诡的地方在于...”或“很多人有个幻觉...”开头。
      - 生活化类比：擅长把复杂技术/金融问题降维打击，翻译成“手术、咖啡、买菜”等日常细节。
  - **段落视觉**：2-4行一断，大量留白，文字密度极低，极其适合手机端滑动阅读。
  - **标志性结尾**：随性收尾，不升华主题，不强行总结（如“就这样，撤了。”、“就聊这么多，下回见。”）。


修辞偏好：
  - **自嘲式金句**：不熬鸡汤，只讲因势利导（如“赌狗哪有什么信仰”、“都是千年的狐狸，别玩什么聊斋”）。
  - **毒舌短评** ：模块化处理热点（事实描述+一句简短犀利的点评）。


示例片段：
> 很多人问我，AI 到底能不能替代程序员。
>
> 其实这个提问最吊诡的地方在于，你把“写代码”等同于“搞产品”了。现在大家都在卷大模型直出二进制，但
这事儿挺有意思的，它就像一个全自动的咖啡机，确实能出咖啡，但它不知道这杯咖啡该端给谁。
>
> 说白了，代码只是表达逻辑的方言，逻辑本身才是值钱的玩意儿。如果你只会翻译，那确实该焦虑；如果你
是那个定口味的人，大可不必。
>
> 行了，就聊这么多，撤了。

## Commands

### /style list
列出所有可用风格模板：
```
可用风格模板：

[1] style-deep-analysis      深度分析型（财新风）
[2] style-casual-interpretation 轻松解读型（36氪风）
[3] style-personal-essay     个人随笔型（公众号大V风）
[4] style-international-perspective 国际视野型（经济学人风）
[5] style-sharp-commentary   犀利评论型（自媒体风）
[6] style-moomoo-casual      理智随性型（猫笔刀风）

[7+] 用户自定义风格...

使用：/style load [编号或名称]
```

### /style load [name]
加载指定风格到当前对话：
```
已加载：style-casual-interpretation（轻松解读型）

特征摘要：
- 短句为主，口语化
- 适合手机阅读，段落2-3行
- 常用过渡："说白了"、"有意思的是"

后续生成将遵循此风格。
```

### /style analyze [text]
分析文本并提取风格指纹：
```
请粘贴你的范文（或你喜欢的文章片段），我将分析其风格特征。
```

**分析输出**：
```
📋 风格分析报告

【句式特征】
- 平均句长：18字（偏短）
- 短句占比：65%（高）
- 特色：爱用设问句、括号补充

【语气特征】
- 人称：第一人称"我"占40%
- 情感：温和中带犀利
- 专业度：中等，术语密度15%

【过渡特征】
- 高频词："你会发现"、"更重要的是"、"说白了"
- 衔接方式：疑问-回答式

【修辞特征】
- 主要手法：比喻（30%）、排比（20%）、故事（25%）
- 标点：破折号使用频率高于常人3倍

【风格标签】
#口语化 #故事驱动 #温和犀利 #手机友好

建议保存为自定义模板？（Y/N）
```

### /style save [name]
将当前分析的风格保存为新模板：
```
已保存为：style-my-voice

下次可直接使用：/style load style-my-voice
```

### /style compare [name1] [name2]
对比两个风格的差异：
```
对比：style-deep-analysis vs style-casual-interpretation

| 维度 | 深度分析型 | 轻松解读型 |
|------|-----------|-----------|
| 句长 | 35字 | 18字 |
| 语气 | 客观冷静 | 轻松幽默 |
| 段落 | 4-5行 | 2-3行 |
| 适合 | 行业报告 | 公众号推文 |
```

## Integration with News Content Factory

在 news-content-factory 的 Step 7 中，当用户输入 `/style` 时：

1. 暂停当前流程
2. 调用 style-library 的交互界面
3. 用户选择或分析风格后，返回风格指纹
4. 继续 Step 7 或直接进入 Step 8

**调用示例**：
```
[Step 7] 请选择写作配置...

User: /style

[style-library 激活]
> /style list
[显示列表]
> /style load 3
[加载个人随笔型]

[style-library 返回风格指纹给 news-content-factory]

[继续 Step 7/8]
已加载风格：个人随笔型
是否继续选择结构？（Y/N）
```

## Custom Style Creation Guide

创建自定义风格的步骤：

1. **收集范文**：找到3-5篇你写的或极度喜欢的文章
2. **粘贴分析**：使用 `/style analyze` 粘贴文本
3. **调整优化**：根据分析结果，告诉AI"再多一些幽默"、"减少长句"等
4. **保存模板**：使用 `/style save [name]`
5. **测试验证**：用同一主题测试不同风格，对比效果

## Style Fingerprint Format

风格指纹的标准格式（供AI内部使用）：

```yaml
style_fingerprint:
  name: "style-name"
  sentence:
    avg_length: 18
    distribution: [short: 0.65, medium: 0.25, long: 0.10]
    patterns: ["rhetorical-questions", "parenthetical-asides"]
  tone:
    person: "first"
    emotion: "warm-with-edge"
    formality: "medium"
  transition:
    common_phrases: ["你会发现", "更重要的是"]
    flow: "question-answer"
  rhetoric:
    primary: ["metaphor", "storytelling"]
    punctuation_pref: ["dash", "ellipsis"]
  paragraph:
    avg_lines_mobile: 2.5
    spacing: "generous"
  vocabulary:
    complexity: "medium"
    jargon_density: 0.15
    loanwords: false
```
