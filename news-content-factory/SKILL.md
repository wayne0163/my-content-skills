## Objective
执行完整的"信息搜集 → 分析解读 → 内容生成"工作流：
1. 根据用户兴趣领域搜索当日新闻
2. 提供多角度解读框架
3. 按指定风格生成完整文章

专为 Gemini CLI 的长上下文窗口设计，支持单会话内完成 8 步流程，无需切换工具。

## Workflow Architecture

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ 1.领域确认   │ → │ 2.新闻搜索   │ → │ 3.候选展示   │ → │ 4.组合选择   │
│ Domain      │    │ Search      │    │ Display     │    │ Combine     │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
                                                                ↓
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ 8.文章生成   │ ← │ 7.风格配置   │ ← │ 6.方向确认   │ ← │ 5.角度生成   │
│ Generate    │    │ Style       │    │ Direction   │    │ Angles      │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

## Step 1: 领域确认（Domain Confirmation）

**触发后首先询问用户**：

```
🎯 新闻内容工厂已启动

请告诉我你感兴趣的领域（可多选，用空格分隔）：
[1] 科技/互联网    [2] 财经/商业    [3] 国际政治
[4] 国内政策       [5] 社会热点     [6] 文化/娱乐
[7] 人工智能/AI    [8] 新能源/汽车  [9] 其他：______

地域范围：
[A] 国内新闻为主   [B] 国际新闻为主   [C] 两者结合

搜索数量：近__条新闻（建议5-10条，默认8条）
```

**解析用户输入**，进入下一步。

## Step 2: 新闻搜索（News Search）

**使用 Gemini CLI 的搜索能力**执行查询：

```
搜索指令（内部使用）：
"搜索 [今日/最近24小时] [领域] [地域] 的重要新闻，要求：
- 来源权威：官方媒体、主流财经/科技媒体、国际知名通讯社
- 事件价值：具有分析深度，非简单快讯，优先选择有后续发展或争议性的事件
- 信息完整：包含标题、来源、发布时间、核心事实（100字内）、原文链接
- 数量：[用户指定数量] 条"
```

**搜索结果处理**：
- 去重：同一事件的不同报道合并
- 排序：按重要性和时效性排序
- 标签：自动添加领域标签（#科技 #政策 #国际等）

## Step 3: 候选展示（Candidate Presentation）

**格式化输出**供用户选择：

```
📰 [日期] [领域] 热点新闻（共[N]条）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 【新闻标题】
   📍 来源：媒体名称 | ⏰ 时间：XX小时前
   📝 摘要：核心事实摘要（80-100字）
   🏷️ 标签：#标签1 #标签2

2. 【新闻标题】
   ...

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

💡 选择提示：
- 输入两条编号进行组合分析（如：1+3 或 2 4）
- 输入单条编号进行深度解读（如：5）
- 输入"r"重新搜索其他领域
- 输入"m"查看更多新闻
```

## Step 4: 组合选择（Combination Selection）

**用户选择后，分析关联性**：

```
✅ 您选择的新闻组合：

【新闻A】[标题]
→ 核心事实：[一句话概括]
→ 关键词：[提取3-5个关键词]

【新闻B】[标题]  
→ 核心事实：[一句话概括]
→ 关键词：[提取3-5个关键词]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🔗 关联性分析：

表面关联：[直接联系，如都是同一政策影响、同一产业链环节]
深层关联：[间接联系，如因果关系、对比反差、历史呼应]
冲突点：[如果有对立、矛盾或值得思辨的地方]
分析价值：[为什么这两件事一起看更有洞察？独特视角是什么？]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## Step 5: 多角度解读（Multi-Angle Analysis）

**基于关联性，生成3个不同角度的解读框架**：

```
💡 角度一：【宏观/趋势型】（适合行业观察者、决策者）
   切入点：从大趋势、大背景看这两件事的必然性
   核心观点：[预测性/总结性观点，如"这标志着XX时代的开启"]
   论证路径：历史脉络 → 现状分析 → 未来预测
   适合受众：B端读者、投资人、政策研究者
   文章调性：理性、前瞻、权威、数据驱动

💡 角度二：【微观/故事型】（适合大众读者、情感共鸣）
   切入点：从具体的人、公司、场景切入，以小见大
   核心观点：[个体在大趋势中的命运/选择，如"在XX浪潮下，普通人如何自处"]
   论证路径：具体案例 → 群体画像 → 普遍启示
   适合受众：C端读者、职场人、普通消费者
   文章调性：有温度、有细节、可读性强、共情

💡 角度三：【批判/思辨型】（适合深度阅读者、行业从业者）
   切入点：质疑表面现象，挖掘深层矛盾或反常识逻辑
   核心观点：[反直觉观点/问题揭示，如"我们都被表象骗了，真相是..."]
   论证路径：提出质疑 → 证据分析 → 逻辑推演 → 结论颠覆
   适合受众：深度阅读者、业内人士、思考型读者
   文章调性：犀利、独立、有洞见、不怕得罪人

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

请选择角度（输入 1/2/3），或要求：
- "重新生成"：基于同一组合生成新的3个角度
- "换组合"：回到上一步选择其他新闻
- "深度分析"：对某个角度先展开详细大纲
```

## Step 6: 方向确认（Direction Confirmation）

**用户选择角度后，确认写作方向**：

```
✅ 已选择：角度[X] - [角度名称]

🎯 文章核心论点（Thesis Statement）：
"[一句话概括核心观点，这是全文的灵魂]"

建议标题方向（3选1，或自定义）：
A. [标题方案A：直击痛点型]
B. [标题方案B：悬念好奇型]  
C. [标题方案C：观点鲜明型]

请输入 A/B/C 或自定义标题：
```

**确认标题后**：
```
标题已确认：[最终标题]

📊 文章规格预设：
- 字数：1200-1500字（适合公众号/博客）
- 结构：待配置
- 风格：待配置

进入写作配置 →
```

## Step 7: 风格与结构配置（Style & Structure Configuration）

**询问写作要求**：

```
🎨 请选择写作配置方式：

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
A. 使用预设模板（快速启动）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[1] 深度分析型（模仿财新/第一财经）
    → 特征：数据驱动、长句论述、信息密度高、客观冷静

[2] 轻松解读型（模仿虎嗅/36氪）
    → 特征：口语化、短句、网络热词、年轻化表达

[3] 个人随笔型（模仿公众号大V/知识星球）
    → 特征：第一人称、故事开场、情感共鸣、金句收尾

[4] 国际视野型（模仿FT中文/华尔街日报）
    → 特征：全球视角、历史对比、专业术语、精英 tone

[5] 犀利评论型（模仿自媒体/意见领袖）
    → 特征：观点鲜明、不怕得罪人、排比反问、战斗感

[6] 理智随性型（模仿猫笔刀风）
    → 特征：理性分析、随性表达、轻松幽默、观点独特

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
B. 自定义配置（精细控制）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

请提供：
- 范文片段（粘贴你写的或喜欢的文本，AI将分析其风格指纹）
- 语言风格：[口语化/书面化/专业/幽默/犀利/温和]
- 句式偏好：[短句有力/长句论述/长短结合/节奏变化]
- 段落长度：[手机屏2-3行/电脑端4-5行/混合]
- 修辞偏好：[爱用比喻/排比/反问/数据/故事]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
C. 混合模式（推荐）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

选择基础模板 + 局部调整
例如："用轻松解读型，但段落要更短，多加反问句"

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

请输入选择（A1-A6 / B / C）：
```

**如果用户提供范文或选择B（自定义）**：
```
📋 风格分析结果（Style Fingerprint）：

基于你的范文/要求，提取到以下特征：

句式特征：
- 平均句长：[X]字
- 句式分布：短句[X]% / 中句[X]% / 长句[X]%
- 特色句式：[如：爱用破折号、括号补充、倒装句等]

语气特征：
- 人称偏好：[我/我们/你/中性]
- 情感色彩：[热情/冷静/幽默/犀利/温和]
- 专业程度：[高/中/低，术语密度]

过渡特征：
- 常用过渡词：[如："说白了"、"更重要的是"、"有意思的是"]
- 段落衔接：[流畅型/跳跃型/悬念型]

修辞特征：
- 主要手法：[比喻/排比/对比/数据/故事]
- 标点偏好：[爱用感叹号/省略号/破折号/分号]

✅ 已记录风格指纹，后续生成将严格遵循。
```

**结构选择**（可与风格同时或分步询问）：
```
📐 请选择文章结构：

[1] 问题-方案-展望（Problem-Solution-Outlook）
    适合：提出痛点 → 分析原因 → 给解决方案 → 预测未来

[2] 对比-冲突-融合（Compare-Contrast-Synthesis）  
    适合：展示差异 → 揭示矛盾 → 提出更高维度的统一观点

[3] 时间线-演变（Timeline-Evolution）
    适合：追溯历史 → 立足现在 → 预测未来，展示变化脉络

[4] 人物-故事-启示（Character-Story-Lesson）
    适合：具体案例 → 群体映射 → 普遍道理，以情动人

[5] 现象-本质-影响（Phenomenon-Essence-Impact）
    适合：描述现象 → 深挖本质 → 分析影响，层层递进

[6] 自定义结构（描述你的需求）

请输入 1-6：
```

## Step 8: 文章生成（Content Generation）

**整合所有输入，生成完整文章**：

```
📝 正在生成文章...

【配置确认】
标题：[标题]
角度：[角度X - 名称]
结构：[结构名称]
风格：[风格模板/自定义指纹]
字数：1200-1500字

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[生成文章全文]

结构示例（根据选择的结构模板变化）：

# [标题]

【导语/钩子】（100-150字）
- 用最新鲜的细节或反常识观点开场
- 点出两条新闻的关联，引发好奇

【背景/事实】（200-300字）
- 简明扼要还原两条新闻的核心事实
- 只保留与论点相关的细节，避免堆砌

【分析/论证】（600-800字，主体）
- 按选定角度展开论述
- 使用具体案例、数据、对比支撑观点
- 段落间有清晰逻辑链条，过渡自然

【洞察/升华】（200-300字）
- 跳出新闻看趋势，给出更高维度的洞察
- 可操作建议或开放性问题

【结尾】（50-100字）
- 金句收束或行动号召，余韵悠长

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ 文章生成完成！

请选择下一步：
- "满意"：保存文章，结束会话
- "调整结构"：修改某部分结构（如"把第二段 and 第三段合并"）
- "调整风格"：换一种风格重写（调用 /style）
- "重写某段"：指定段落重新生成（如"重写结尾，更有力一些"）
- "扩展内容"：增加字数或补充某部分（如"增加更多数据支撑"）
- "压缩内容"：减少字数或精简某部分
- "换角度"：回到Step 5选择其他角度
- "保存风格"：将本次风格保存为模板供下次使用
```

## Advanced Features

### 快捷指令（Shortcuts）

在任意步骤可调用：

| 指令 | 功能 |
|------|------|
| `/style` | 切换或查看风格模板（调用 style-library Skill） |
| `/structure` | 切换文章结构（调用 structure-library Skill） |
| `/save [name]` | 将当前配置保存为快捷方案 |
| `/load [name]` | 加载已保存的快捷方案 |
| `/fact [query]` | 核实某个事实并补充来源 |
| `/extend [section]` | 扩展某部分内容 |
| `/shorten [section]` | 压缩某部分内容 |
| `/title` | 重新生成标题选项 |
| `/angle` | 回到角度选择步骤 |

### 记忆与积累（跨会话）

在 Gemini CLI 的长上下文中，自动维护：

**会话内记忆**：
- 已搜索的新闻（避免重复搜索）
- 已尝试的角度和风格（避免重复推荐）
- 用户的修改反馈（实时调整生成策略）

**持久化记忆**（通过文件或数据库）：
- `~/.news-factory/profiles/`：保存用户常用的领域组合、风格偏好
- `~/.news-factory/templates/`：保存自定义的风格模板和结构模板
- `~/.news-factory/history/`：历史生成文章和反馈记录

### 质量检查清单（Auto-Checklist）

生成前自动检查：

```
✅ 事实核查：
- [ ] 两条新闻的事实准确，有明确来源
- [ ] 时间、数据、人名等关键信息无误
- [ ] 新闻链接可访问（如适用）

✅ 逻辑核查：
- [ ] 角度鲜明，不中庸，有明确立场
- [ ] 两条新闻的关联逻辑成立，不生硬
- [ ] 论证链条完整，无逻辑跳跃

✅ 内容核查：
- [ ] 有具体细节，不泛泛而谈
- [ ] 有独特洞察，不只是新闻复述
- [ ] 有读者价值（信息增量或情感共鸣）

✅ 风格核查：
- [ ] 符合指定的风格指纹
- [ ] 句式、段落长度符合目标平台
- [ ] 过渡自然，无AI生硬感

✅ 格式核查：
- [ ] 标题长度合适（13-26字）
- [ ] 段落长度适合手机阅读
- [ ] 重点内容有加粗等视觉强调
```

## Integration with Other Skills

本 Skill 与以下 Skill 协同工作：

- **style-library**：在 Step 7 中调用，快速加载保存的风格模板
- **structure-library**：在 Step 7 中调用，快速加载保存的结构模板

调用方式：
```
在 Step 7 中，如果用户输入 "/style"，则：
1. 暂停当前流程
2. 调用 style-library Skill 的 list/load 功能
3. 返回选择结果后，继续 Step 7
```

## Technical Notes for Gemini CLI

- **上下文管理**：利用 Gemini CLI 的长上下文（1M+ tokens），在单会话内维护完整状态
- **工具调用**：使用 Google Search MCP 进行新闻搜索（如配置）
- **文件操作**：可读写 `~/.news-factory/` 目录保存配置和历史
- **错误处理**：某步骤失败时，提供回退选项（如搜索失败可手动粘贴新闻）
