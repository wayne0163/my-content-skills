# Article Structure Library (Detailed Frameworks)

## S1: 问题-方案-展望 (Problem-Solution-Outlook)
**适用**：提出痛点、分析原因、给解决方案、预测未来
**逻辑**：Hook -> Problem -> Analysis -> Solution -> Outlook

**框架详解**：
1. **【Hook/痛点场景】**（100-150字）
   - 一个具体的、让人共鸣的痛点场景；"你有没有遇到过..."
2. **【Problem/问题阐述】**（200-300字）
   - 问题是什么？（用新闻事实说明）；问题的严重性和影响范围；为什么现在必须解决？
3. **【Analysis/原因分析】**（300-400字）
   - 为什么会这样？深层原因是什么？历史脉络或对比案例；涉及的利益相关方。
4. **【Solution/解决方案】**（300-400字）
   - 已有的解决方案及其局限；更好的解决方案（你的观点）；具体可操作的步骤。
5. **【Outlook/未来展望】**（150-200字）
   - 如果实施了解决方案会怎样？潜在的风险和挑战；对读者的意义和行动建议。

---

## S2: 对比-冲突-融合 (Compare-Contrast-Synthesis)
**适用**：展示差异、揭示矛盾、提出更高维度的统一观点
**逻辑**：Hook -> Contrast -> Connection -> Conflict -> Synthesis

**框架详解**：
1. **【Hook/两个看似无关的事】**（100-150字）
   - 抛出两条新闻或两个现象；"表面上看，这两件事毫不相干..."
2. **【Contrast/对比展示】**（300-400字）
   - 详细展示两者的不同之处；A与B的特点、维度（时间/空间/方式等）对比。
3. **【Connection/隐藏关联】**（200-300字）
   - 挖掘深层的联系；共同的驱动因素或背景；"但如果我们把视野拉远..."
4. **【Conflict/矛盾或张力】**（250-350字）
   - 揭示冲突点或悖论；为什么两者不能共存？或为什么同时出现是荒谬的？
5. **【Synthesis/融合与超越】**（300-400字）
   - 提出更高维度的解释框架；新的认知模型或解决方案；对读者的启示。

---

## S3: 时间线-演变 (Timeline-Evolution)
**适用**：追溯历史、立足现在、预测未来，展示变化脉络
**逻辑**：Present -> Past -> Present -> Future -> Meaning

**框架详解**：
1. **【Present/现在的热点】**（100-150字）
   - 从最新的热点事件切入；"最近，XX事件引发了广泛关注..."
2. **【Past/历史回顾】**（300-400字）
   - 这件事是怎么来的？关键历史节点（3-5个）；每个节点的变化和影响。
3. **【Present/现在的位置】**（250-350字）
   - 现在处于什么阶段？与历史相比的本质不同；当前的矛盾和挑战。
4. **【Future/未来预测】**（250-350字）
   - 接下来会发生什么？（短中长期）；基于趋势的逻辑推演。
5. **【Meaning/意义与启示】**（150-200字）
   - 对普通人的意义；我们应该如何应对？历史给我们的教训。

## S4: 人物-故事-启示 (Character-Story-Lesson)
**适用**：具体案例、群体映射、普遍道理，以情动人
**逻辑**：Character -> Story -> Context -> Insight -> Lesson

**框架详解**：
1. **【Character/人物登场】**（150-200字）
   - 一个具体的人/公司/组织；背景介绍：是谁？在什么处境？
2. **【Story/故事展开】**（400-500字）
   - 遇到的挑战（结合新闻）；做的选择；过程细节（关键转折点）；结果如何？
3. **【Context/群体映射】**（200-300字）
   - 这个案例不是孤例，代表了一个群体或现象；数据支撑。
4. **【Insight/背后的道理】**（250-350字）
   - 这个故事告诉我们什么？深层的社会/经济/心理机制。
5. **【Lesson/给读者的启示】**（150-200字）
   - 对普通人的可操作建议；情感共鸣或行动号召。

---

## S5: 现象-本质-影响 (Phenomenon-Essence-Impact)
**适用**：描述现象、深挖本质、分析影响，层层递进
**逻辑**：Phenomenon -> Surface -> Essence -> Impact -> Response

**框架详解**：
1. **【Phenomenon/现象描述】**（200-300字）
   - 详细描述看到的现象（结合新闻）；表面特征：发生了什么？有何异常？
2. **【Surface/表层解释】**（150-200字）
   - 大多数人怎么看（主流观点）；为什么这个解释不够？
3. **【Essence/本质挖掘】**（400-500字）
   - 真正的驱动力是什么？剥开表象（利益、权力、心理、技术等因素）。
4. **【Impact/影响分析】**（300-400字）
   - 对行业、个人、社会的长期影响；连锁反应。
5. **【Response/应对与思考】**（150-200字）
   - 面对这一本质，我们该怎么办？个人层面的应对建议。

---

## S6: 问题-证据-结论 (Question-Evidence-Conclusion)
**适用**：学术研究、调查报告、事实核查，严谨论证
**逻辑**：Question -> Hypothesis -> Evidence -> Analysis -> Conclusion

**框架详解**：
1. **【Question/核心问题】**（100-150字）
   - 提出一个具体、可验证的问题；为什么这个问题重要？
2. **【Hypothesis/假设提出】**（100-150字）
   - 基于现有信息的初步假设；可能的多重假设。
3. **【Evidence/证据收集】**（400-600字）
   - 多个证据的来源、内容、可靠性评估；对比不同证据的矛盾。
4. **【Analysis/逻辑推演】**（200-300字）
   - 基于证据的逻辑分析（排除法、归纳法等）；考虑反方观点。
5. **【Conclusion/结论与局限】**（150-200字）
   - 明确的结论；置信度评估；未来验证方向。
