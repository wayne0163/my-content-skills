# xunxun-shipin-content

此技能旨在为 30-55 岁、有资产、有风险意识但非专业的读者制作高质量的短视频口播文稿。核心逻辑在于“用专业度拆解复杂金融，用透明度建立信任”。

## 核心原则
- **目标读者**：30-55岁非专业人群（有资产，有风险意识）。
- **语言风格**：分层表达、立场判断、通俗翻译、克制语气。
- **数据驱动**：使用具体数据（IRR、评级、资产占比）而非虚词。
- **透明度**：主动说明缺点（自曝其短），界定确定性与不确定性。

## 工作流

1. **信息收集**：
   - 必须要求用户提供：**视频时长**（如1分钟、3分钟）和**核心信息/资料**（产品条款、案例数据等）。
   - 询问用户是否需要切换风格或结构（默认使用“专业坦诚型”风格和“经典五步法”结构）。

2. **标题建议**：
   - 综合用户提供的信息，先生成 3-5 个具有吸引力的标题供用户选择。

3. **文稿生成与迭代**：
   - 根据选定的时长和参考文件中的定义生成文稿。
   - 参考 [styles.md](references/styles.md) 获取风格要求，参考 [structures.md](references/structures.md) 获取结构建议。
   - 确保文稿包含视觉化语言（如“看屏幕这个位置”）和具体案例建模。
   - **满意度确认**：生成文稿后，必须询问用户：“您对这份文稿满意吗？如果有需要调整的地方，请告诉我也。”
   - **循环修改**：根据用户的修改意见持续优化文稿，并再次询问，直到得到用户“满意”、“好”或“保存”的明确回复。

4. **自动保存**：
   - 在用户确认满意或要求保存后，使用 `write_file` 工具在当前目录保存最终版。
   - **文件命名规范**：`yyyy-mm-dd+标题.md`（例如：2026-02-17+养老保险深度拆解.md）。

## 使用建议
- **1分钟视频**：重点在于快速切入问题并给出核心拆解。
- **3分钟或以上视频**：允许更深度的公司底层实力分析和场景化理赔模拟。
