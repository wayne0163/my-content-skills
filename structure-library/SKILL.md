## Objective
提供标准化的文章结构模板，支持：
1. 快速选择适合的论证框架
2. 自定义文章结构
3. 为 news-content-factory 提供结构配置
4. 确保文章逻辑清晰、层次分明

## Available Templates

### structure-problem-solution（问题-方案-展望）
**适用**：提出痛点、分析原因、给解决方案、预测未来
**经典场景**：产品分析、政策解读、行业痛点

**框架**：
```
1. 【Hook/痛点场景】（100-150字）
   - 一个具体的、让人共鸣的痛点场景
   - "你有没有遇到过..."

2. 【Problem/问题阐述】（200-300字）
   - 问题是什么？（用新闻事实说明）
   - 问题的严重性和影响范围
   - 为什么现在必须解决？

3. 【Analysis/原因分析】（300-400字）
   - 为什么会这样？深层原因是什么？
   - 历史脉络或对比案例
   - 涉及的利益相关方

4. 【Solution/解决方案】（300-400字）
   - 已有的解决方案及其局限
   - 更好的解决方案（你的观点）
   - 具体可操作的步骤

5. 【Outlook/未来展望】（150-200字）
   - 如果实施了解决方案会怎样？
   - 潜在的风险和挑战
   - 对读者的意义和行动建议
```

**特点**：逻辑闭环，从痛点到希望，适合说服性文章

### structure-compare-contrast（对比-冲突-融合）
**适用**：展示差异、揭示矛盾、提出更高维度的统一观点
**经典场景**：两条新闻对比、新旧模式对比、中外对比

**框架**：
```
1. 【Hook/两个看似无关的事】（100-150字）
   - 抛出两条新闻或两个现象
   - "表面上看，这两件事毫不相干..."

2. 【Contrast/对比展示】（300-400字）
   - 详细展示两者的不同之处
   - A的特点是什么？B的特点是什么？
   - 对比维度：时间、空间、主体、方式、结果

3. 【Connection/隐藏关联】（200-300字）
   - 挖掘深层的联系
   - 共同的驱动因素或背景
   - "但如果我们把视野拉远..."

4. 【Conflict/矛盾或张力】（250-350字）
   - 揭示冲突点或悖论
   - 为什么两者不能共存？
   - 或：为什么同时出现是荒谬的？

5. 【Synthesis/融合与超越】（300-400字）
   - 提出更高维度的解释框架
   - 不是A对B错，而是...
   - 新的认知模型或解决方案
   - 对读者的启示
```

**特点**：思辨性强，适合深度分析和观点输出

### structure-timeline-evolution（时间线-演变）
**适用**：追溯历史、立足现在、预测未来，展示变化脉络
**经典场景**：行业变迁、技术演进、政策变化、个人成长

**框架**：
```
1. 【Present/现在的热点】（100-150字）
   - 从最新的热点事件切入
   - "最近，XX事件引发了广泛关注..."

2. 【Past/历史回顾】（300-400字）
   - 这件事是怎么来的？
   - 关键的历史节点（3-5个）
   - 每个节点的变化和影响
   - "让我们把时间拨回到X年前..."

3. 【Present/现在的位置】（250-350字）
   - 现在处于什么阶段？
   - 与历史相比，有什么本质不同？
   - 当前的矛盾和挑战

4. 【Future/未来预测】（250-350字）
   - 接下来会发生什么？（短中长期）
   - 基于趋势的逻辑推演
   - 可能的分支和变数

5. 【Meaning/意义与启示】（150-200字）
   - 对普通人的意义
   - 我们应该如何应对？
   - 历史给我们的教训
```

**特点**：纵深感强，适合展现变化规律和趋势判断

### structure-character-story-lesson（人物-故事-启示）
**适用**：具体案例、群体映射、普遍道理，以情动人
**经典场景**：创业者故事、职场案例、社会现象、个人经历

**框架**：
```
1. 【Character/人物登场】（150-200字）
   - 一个具体的人（或公司、组织）
   - 背景介绍：是谁？在什么处境？
   - "XX，今年XX岁，是..."

2. 【Story/故事展开】（400-500字）
   - 遇到了什么挑战？（结合新闻背景）
   - 做了什么选择？
   - 过程和细节（关键转折点2-3个）
   - 结果如何？
   - 有对话、有场景、有心理描写

3. 【Context/群体映射】（200-300字）
   - 这个人不是孤例
   - 代表了一个群体或一种现象
   - 数据支撑：这样的人有多少？趋势如何？

4. 【Insight/背后的道理】（250-350字）
   - 这个故事告诉我们什么？
   - 深层的社会/经济/心理机制
   - 与新闻背景的关联

5. 【Lesson/给读者的启示】（150-200字）
   - 对普通人的可操作建议
   - 情感共鸣或行动号召
   - "如果你也面临类似处境，不妨..."
```

**特点**：可读性强，情感共鸣，适合大众传播

### structure-phenomenon-essence-impact（现象-本质-影响）
**适用**：描述现象、深挖本质、分析影响，层层递进
**经典场景**：社会热点、新趋势解读、异常事件分析

**框架**：
```
1. 【Phenomenon/现象描述】（200-300字）
   - 详细描述你看到的现象（结合新闻）
   - 表面特征：发生了什么？有什么异常？
   - "最近，一个有趣的现象引起了注意..."

2. 【Surface/表层解释】（150-200字）
   - 大多数人怎么看？（主流观点）
   - 为什么这个解释不够？

3. 【Essence/本质挖掘】（400-500字）
   - 真正的驱动力是什么？
   - 打破表象，揭示深层机制
   - 涉及的利益、权力、心理、技术等因素
   - "但如果我们剥开表象，会发现..."

4. 【Impact/影响分析】（300-400字）
   - 对行业的影响
   - 对个人的影响
   - 对社会的长期影响
   - 连锁反应和蝴蝶效应

5. 【Response/应对与思考】（150-200字）
   - 面对这一本质，我们该怎么办？
   - 个人层面的应对
   - 社会层面的建议
   - 开放性问题留给读者
```

**特点**：深度挖掘，洞察导向，适合建立专业权威

---

### structure-question-evidence-conclusion（问题-证据-结论）
**适用**：学术研究、调查报告、事实核查，严谨论证
**经典场景**：科普文章、辟谣、深度调查

**框架**：
```
1. 【Question/核心问题】（100-150字）
   - 提出一个具体、可验证的问题
   - 为什么这个问题重要？
   - "XX真的是这样吗？"

2. 【Hypothesis/假设提出】（100-150字）
   - 基于现有信息的初步假设
   - 可能的多重假设

3. 【Evidence/证据收集】（400-600字）
   - 证据1：来源、内容、可靠性评估
   - 证据2：来源、内容、可靠性评估
   - 证据3：来源、内容、可靠性评估
   - 对比不同证据的矛盾和一致性

4. 【Analysis/逻辑推演】（200-300字）
   - 基于证据的逻辑分析
   - 排除法、归纳法、演绎法
   - 考虑反方观点和反驳

5. 【Conclusion/结论与局限】（150-200字）
   - 明确的结论
   - 置信度评估
   - 证据局限和未来验证方向
```

**特点**：严谨客观，证据驱动，适合建立可信度

## Commands

### /structure list
列出所有可用结构模板：
```
可用结构模板：

[1] structure-problem-solution      问题-方案-展望
[2] structure-compare-contrast      对比-冲突-融合
[3] structure-timeline-evolution    时间线-演变
[4] structure-character-story       人物-故事-启示
[5] structure-phenomenon-essence    现象-本质-影响
[6] structure-question-evidence     问题-证据-结论

[7+] 用户自定义结构...

使用：/structure [编号或名称]
```

### /structure [name]
应用结构到当前文章：
```
已选择：structure-compare-contrast（对比-冲突-融合）

此结构要求：
- 两条新闻或两个现象
- 展示差异 → 揭示矛盾 → 提出统一观点
- 适合思辨性分析

是否查看详细大纲？（Y/N）
```

### /structure custom [description]
根据描述生成自定义结构：
```
请描述你需要的文章结构：
> 我想先讲一个故事，然后分析背后的数据，最后给出建议

生成自定义结构：

【自定义结构：故事-数据-建议】

1. 【Story/故事引入】
   - 具体案例或场景
   - 情感共鸣或悬念设置

2. 【Data/数据分析】
   - 故事背后的数据支撑
   - 宏观趋势与微观案例的关联

3. 【Advice/建议给出】
   - 基于故事和数据的建议
   - 可操作的步骤

是否保存此结构？（Y/N）
```

### /structure preview [name]
预览指定结构的详细大纲：
```
structure-compare-contrast 详细大纲：

[每个部分的字数建议、写作要点、过渡提示]
```

## Integration with News Content Factory

在 news-content-factory 的 Step 7 中，当用户输入 `/structure` 时：

1. 暂停当前流程
2. 调用 structure-library 的交互界面
3. 用户选择结构后，返回结构框架
4. 继续 Step 7 或直接进入 Step 8

**调用示例**：
```
[Step 7] 请选择写作配置...

User: /structure

[structure-library 激活]
> /structure list
[显示列表]
> /structure 2
[选择对比-冲突-融合结构]

[structure-library 返回结构框架给 news-content-factory]

[继续 Step 7/8]
已选择结构：对比-冲突-融合
是否继续选择风格？（Y/N）
```

## Structure Selection Guide

如何选择适合的结构？

| 如果你有... | 选择结构 |
|------------|---------|
| 一个痛点想解决 | 问题-方案-展望 |
| 两条新闻要对比 | 对比-冲突-融合 |
| 一个变化要解读 | 时间线-演变 |
| 一个案例要讲 | 人物-故事-启示 |
| 一个现象要深挖 | 现象-本质-影响 |
| 一个疑问要查证 | 问题-证据-结论 |

## Custom Structure Creation

创建自定义结构的步骤：

1. **明确需求**：你的文章要解决什么问题？
2. **拆解模块**：需要哪几个部分？每部分的功能？
3. **逻辑排序**：模块之间的先后顺序和过渡关系
4. **字数分配**：每部分大概多少字？比例如何？
5. **保存模板**：使用 `/structure custom` 描述并保存
