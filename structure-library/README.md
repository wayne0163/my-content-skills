# Structure Library

文章结构库：标准化写作框架，让逻辑更清晰。

## 为什么需要结构模板？

同样的内容，不同的结构，效果天差地别：
- 结构清晰 → 读者跟得上你的思路
- 结构混乱 → 读者中途放弃

## 6大核心结构

| 结构 | 核心逻辑 | 最适合 |
|------|---------|--------|
| 问题-方案-展望 | 痛点→解决→未来 | 产品分析、政策解读 |
| 对比-冲突-融合 | 差异→矛盾→超越 | 两条新闻对比、新旧模式 |
| 时间线-演变 | 过去→现在→未来 | 行业变迁、技术演进 |
| 人物-故事-启示 | 个体→群体→道理 | 案例故事、社会现象 |
| 现象-本质-影响 | 表象→深层→后果 | 热点解读、趋势分析 |
| 问题-证据-结论 | 疑问→验证→答案 | 调查报道、科普辟谣 |

## 使用方式

### 独立使用

```bash
> /structure list              # 查看所有结构
> /structure 2                 # 选择第2个结构
> /structure custom [描述]      # 创建自定义结构
```

### 在新闻工作流中使用

在 `news-content-factory` 的 Step 7 中，输入 `/structure` 即可调用本库。

## 结构 vs 风格

- **结构（Structure）**：文章的骨架，逻辑框架，"先写什么后写什么"
- **风格（Style）**：文章的血肉，语言表达，"用什么语气写"

两者独立，可任意组合：
- 对比结构 + 犀利风格 = 观点交锋文
- 故事结构 + 轻松风格 = 可读性强的案例文
- 时间线结构 + 国际视野风格 = 深度趋势分析

## 创建自定义结构

如果6个模板都不满足：

```bash
> /structure custom
> 我想先讲一个反常识的观点，然后用数据证明，最后讲对个人的启示
[AI生成自定义结构]
> 保存为：反常识-数据-启示结构
```

## 每个结构包含

- 适用场景说明
- 详细大纲（每部分字数、内容要点）
- 过渡提示（如何衔接各部分）
- 示例文章（可选）

让AI写作有章可循，不再散乱。
